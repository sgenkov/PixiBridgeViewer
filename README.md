# Pixi Bridge (Socket + PixiJS Viewer)

Тази папка съдържа малка, самостоятелна настройка:

- `server/` - Node.js WebSocket сървър, който препраща обновленията.
- `client/` - PixiJS изглед, който визуализира входящите параметри.

Идеята е сървърът да се пуска локално и да се подава данни от проекта за слот игри.

## Бърз старт

1) Инсталирай всички зависимости наведнъж (client + server) от корена:

```
cd pixi-bridge
npm install
```

2) Стартирай всичко (watch на клиента + автоматичен рестарт на сървъра):

```
cd pixi-bridge
npm run dev
```

3) Отвори Pixi клиента:

```
http://localhost:8787
```

4) Изпрати тестови данни:

```
node server/sender-example.js
```

## Формат на съобщенията

Сървърът приема JSON през WebSocket и го разпраща към всички свързани клиенти.

Пример:

```json
{
  "type": "paramsUpdate",
  "payload": {
    "mode": "MEGA_FREE_SPINS",
    "multiplier": 14,
    "reel": 2,
    "symbol": "scatter"
  },
  "ts": 1700000000
}
```

## Бележки

- Скриптът е умишлено минимален и не пипа съществуващия game код.
- Можеш да се свържеш към `ws://localhost:8787/ws` от проекта за слотове, когато е готово.
